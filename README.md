TicketHub est une API REST de billetterie événementielle construite avec Flask (Python 3.10+), suivant le patron Application Factory pour une architecture modulaire basée sur des blueprints (auth, événements, réservations, paiements, admin). Le système offre une authentification complète par jetons JWT (inscription, connexion, rafraîchissement, déconnexion) avec contrôle d'accès par rôle (utilisateur/admin), un CRUD complet pour la gestion des événements avec calcul dynamique des places restantes, un système de réservation supportant la sélection interactive de sièges et l'admission générale par quantité, ainsi qu'un flux de simulation de paiement avec suivi de statut (en attente → confirmé → échoué) et génération de numéros de transaction. La couche de données repose sur SQLAlchemy comme ORM avec MySQL 8.0, les migrations versionnées sont gérées par Flask-Migrate (Alembic), et le schéma comprend cinq tables liées : users, events, seats, bookings et payments. La sécurité est renforcée par Flask-Talisman (Content-Security-Policy, Strict-Transport-Security, X-Content-Type-Options, X-Frame-Options, Referrer-Policy), le hachage des mots de passe via Werkzeug (pbkdf2:sha256), une liste de blocage de jetons révoqués, la configuration CORS par origines autorisées, la validation des entrées et la prévention d'injection SQL par requêtes paramétrées. Un tableau de bord administrateur protégé permet la gestion des utilisateurs, des réservations et la consultation de statistiques (revenus, totaux). Pour démarrer, cloner le dépôt, créer un environnement virtuel, installer les dépendances via pip, configurer le fichier .env avec les clés SECRET_KEY, DATABASE_URL, JWT_SECRET_KEY, CORS_ORIGINS et TALISMAN_FORCE_HTTPS, créer la base de données MySQL, puis exécuter les migrations avec flask db init, flask db migrate et flask db upgrade. Les tests sont exécutés via pytest avec couverture et couvrent l'authentification, les événements, les réservations, les paiements et les routes admin.
